# TreeChat (暂定名) - 愿景与功能设计方案

## 1. 项目简介

TreeChat 旨在打破传统 AI 对话的“线性锁定”，将聊天室从传统的“单一链表”模式进化为“多分支树状结构”。用户可以在对话的任何节点开启平行分支，自由探索不同的逻辑可能性，并最终将多个分支的智慧合并为一个结论。

## 2. 核心痛点

- **上下文污染**：后续对话常受前面无关信息干扰。
- **思维受限**：无法在同一上下文中并行测试不同方向。
- **难以收敛**：散乱的对话难以汇总成结构化的成果。

## 3. 核心功能 (MVP)

### 3.1 分支探索 (Forking)

- **任意点分叉**：用户可以点击对话历史中的任意一条消息，以此为起点开启新的对话分支。
- **平行对比**：在不同的分支中可以设置不同的 System Prompt 或调用不同的 AI 模型。

### 3.2 节点管理 (Node Management)

- **剪枝 (Pruning)**：一键移除不符合预期的对话分支及其子节点。
- **折叠与展开**：保持工作区整洁，只关注当前活跃的思路。

### 3.3 节点合并 (Merging)

- **多选合并**：允许用户选择多个非连续或不同分支的节点。
- **智能综述**：AI 会自动提取所选节点的核心内容，生成一份综合上下文，作为新分支的起始节点。

### 3.4 交互设计

- **无限画布**：类似思维导图的缩放与拖拽体验。
- **视觉导航**：通过缩略图或层级树状图快速定位复杂的对话脉络。
- **全局安全防护**：提供全局且优雅的消息提示 (Toaster) 和危险操作容错与拦截机制 (Confirm Dialog)，防止误删珍贵的对话树资产。

### 3.5 数据主权与模型解耦

- **一切以数据为源**：对话树生成的所有内容（节点流、状态、提示词）均为开放的结构化数据格式，支持全量的**导出与导入**。方便用户备份对话资产、更换设备同步或分享“思维模板”给他人。
- **自由切换各类 AI 端点**：在底层完全解耦特定大模型厂商的 API 锁定。用户可以在全局或在某一个具体分支节点处，随时切换到各种不同的 AI 端点（例如：OpenAI, DeepSeek, Claude, Gemini，甚至是本地运行的 Ollama），同时还可以极其方便地在平行的多个分支下横向对比验证不同 AI 模型的回答能力。

## 4. 技术栈建议 (初期)

- **前端**：React + React Flow (强大的流程图库) 或 Canvas/SVG 自研。
- **状态管理**：Zustand 或 Redux (处理复杂的树状 JSON 数据结构)。
- **后端**：轻量级 Node.js/Python 转发，支持各类 LLM API (DeepSeek, OpenAI 等)。

## 5. 开源计划

- **首选平台**：GitHub
- **目标**：提供一套标准的多分支 AI 对话 UI 组件库和数据协议。
